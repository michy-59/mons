# 問題データの管理手順

## 概要

問題データは CSV ファイルで管理します。
作成 → 検証 → Git アップロード の順で作業します。

---

## 1. CSV ファイルの構成

以下のカラムで CSV を作成してください。

| カラム名 | 内容 |
|----------|------|
| `year`   | 問題に対応する年 |
| `no`     | 同じ `year` 内での順番（目安） |
| `prob`   | 問題文 |
| `ans`    | 解答 |
| `rem`    | 備考（任意） |

---

## 2. 問題の作成

各カラムを以下のルールで記入します。

1. **`year` / `no`**　数値を記入する
2. **`prob`（問題文）**　解答にあたる部分を `**` で置き換える
   - 例：`藤原内麻呂の子について、長男は真夏、次男は冬嗣、三男は秋継、四男は**である`
3. **`ans`（解答）**　`prob` の `**` に対応する答えを記入する
4. **`rem`（備考）**　補足情報があれば記入する。不要なら空欄でよい

---

## 3. アップロード前の検証

以下のスクリプトを実行して、データに問題がないか確認します。

```bash
python3 scr/rearrange.py
```

### ❌ 重複がある場合

`year` と `no` の組み合わせに重複があると、以下のように表示されます。

```
   year  no  cnt
0  1038   1    2
1  1051   2    3
```

`cnt` が重複数を示します。該当箇所の `no` を修正して、`year` + `no` の組み合わせが一意になるようにしてください。

### ✅ 問題がない場合

```
OK
234
```

`OK` の下の数字が作成済みの問題数です。

> **注意：** このスクリプトの実行により、`prob` 列の `**` は半角スペース 4 つを含む半角括弧 `(    )` に自動置換されます。

---

## 4. Git へのアップロード

```bash
./push.sh
```

- コミットコメントは問題数 `N` に応じて自動的に `N mons` となります。
- `scr/create_view.py` も実行されて `view/` ディレクトリに、入力 CSV と同名の Markdown ファイルが生成されます。
  - 最左列に連番 `#` が付いた表形式で出力されます。
  - 半角スペースもそのまま反映されます。
